## Abstract

Hospital readmission within 30 days of discharge is a major quality indicator and cost driver in the US healthcare system. This study applies supervised machine learning to the publicly available UCI Diabetes 130-US Hospitals dataset (N ≈ 71,518 unique patients, 50 features) to predict early readmission among diabetic patients. We compare four models — Logistic Regression, Random Forest, Gradient Boosting, and XGBoost — using a patient-aware train/validation/test split to prevent data leakage. Bootstrap 95% confidence intervals are reported for all primary metrics. SHAP values are used to explain model predictions, and subgroup performance is analysed across race, age, and gender. **Keywords:** hospital readmission, diabetes, machine learning, clinical informatics, algorithmic fairness, SHAP, XGBoost

## 1. Background

Diabetes mellitus affects approximately 537 million adults worldwide and is a leading cause of hospitalisation. In the United States, approximately 20% of hospitalised diabetic patients are readmitted within 30 days of discharge, at an estimated cost of $26 billion annually. Since 2012, the Centers for Medicare and Medicaid Services (CMS) have penalised hospitals with excess readmission rates under the Hospital Readmissions Reduction Program (HRRP).

Predictive models that identify high-risk patients at discharge could enable targeted interventions — enhanced discharge education, earlier follow-up appointments, or dedicated care coordination — to reduce preventable readmissions.

## 2. Dataset

- **Source:** UCI Machine Learning Repository — Diabetes 130-US Hospitals (Strack et al., 2014)
- **URL:** https://archive.ics.uci.edu/dataset/296/diabetes+130-us+hospitals+for+years+1999-2008
- **Period:** 1999–2008, 130 US hospitals
- **Raw dataset:** 101,766 encounters × 50 features
- **After cleaning:** ~71,518 unique patients (first encounter only)
- **Target:** Binary — early readmission within 30 days (1) vs. not (0)
- **Class balance:** ~89% negative, ~11% positive

### Key features
Clinical and administrative features include: patient demographics (age, sex, race), admission/discharge metadata, laboratory result counts, procedure counts, medication counts, 24 individual diabetes medication columns, HbA1c and glucose serum results, and three ICD-9 diagnosis codes (grouped into 9 disease categories).

## 3. Methods

### 3.1 Data Cleaning
- Replaced `?` with NaN; dropped `weight` (97% missing), `payer_code` (40%), `medical_specialty` (49%)
- Removed encounters with hospice/expired discharge disposition
- Removed encounters with Invalid/Unknown gender
- Retained first encounter per patient to prevent patient-level leakage

### 3.2 Feature Engineering
- ICD-9 codes → 9 disease category groups
- Age bins → ordinal integer (0–9)
- 24 medication columns → ordinal encoding (No=0, Steady=1, Down=2, Up=3)
- Derived: `n_medications_active`, `n_medication_changes`, `total_prior_visits`, `had_prior_inpatient`, `primary_diag_is_diabetes`
- A1C result and glucose serum → ordinal + binary tested flag

### 3.3 Data Splitting
Patient-aware GroupShuffleSplit: 70% train / 15% val / 15% test by unique patient ID.
Verified zero patient overlap across all three splits.

### 3.4 Preprocessing
- log1p transform for right-skewed utilisation features
- StandardScaler fitted on training data only
- SMOTE applied to training data to address class imbalance

### 3.5 Models
| Model | Key Hyperparameters |
|---|---|
| Logistic Regression | C=1.0, class_weight='balanced', lbfgs |
| Random Forest | 300 trees, max_depth=10, min_samples_leaf=20 |
| Gradient Boosting | 300 estimators, max_depth=4, lr=0.05 |
| XGBoost | 300 estimators, max_depth=4, lr=0.05, scale_pos_weight |

### 3.6 Evaluation
- Primary metric: ROC-AUC (insensitive to threshold choice)
- Secondary: Average Precision / PR-AUC (appropriate for imbalanced data)
- Clinical: Recall for class 1 (minimise missed readmissions)
- Bootstrap 95% CI: 1,000 resamples
- McNemar's test for pairwise model comparison

## 5. Error Analysis

False Negative analysis (Notebook 11) suggests that missed readmissions tend to occur in patients with fewer prior inpatient admissions — making them appear lower-risk at discharge. A lower classification threshold (e.g. 0.35–0.40) would capture more readmissions at the cost of increased false alarms.

---

## 6. Limitations

1. **Temporal gap:** Data from 1999–2008; clinical practice has changed substantially.
2. **Single-system readmission:** Only returns to the same hospital system are captured.
3. **Class imbalance:** ~11% positive rate limits absolute recall.
4. **No external validation:** Results reflect performance within the same dataset distribution.
5. **Administrative labels:** "Early readmission" includes planned readmissions, which may inflate the target.
6. **Missing features:** Many clinically important features (e.g. socioeconomic status, discharge destination, patient adherence history) are absent.

- Model should be used as a screening aid, not a decision-maker
